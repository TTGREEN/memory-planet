# OpenClaw 水产市场概览

当用户询问“OpenClaw 水产市场是什么”“它和资产体系是什么关系”“为什么市场里会有多种资产类型”时，读取本文件。

本文件只提供平台背景与概念框架。若用户要执行具体操作，返回 `SKILL.md` 中的 CLI 工作流；若用户要区分资产类型，读取 `references/asset-types.md`。

## 平台定位

OpenClaw 水产市场是 OpenClaw 生态中的资产分发、发现和管理平台，用于让用户和 agent 搜索、安装、发布和组织不同类型的能力资产。

可将其理解为面向 agent 能力组件的市场：

- 用户或 agent 在这里发现可用资产
- 通过 CLI 或平台流程安装、发布和管理资产
- 不同资产类型承担不同的系统角色

## 为什么会有资产分类

水产市场中的资产类型，主要按“系统角色”区分，而不是按“技术复杂度”区分。

同一个资产可能技术上很简单或很复杂，但它在系统中的核心职责通常更重要，例如：

- 教 agent 如何完成任务
- 给 agent 增加新的工具调用能力
- 在事件发生时自动唤起 agent
- 让 agent 接入某个消息平台
- 沉淀经验方案、模板或配置思路

因此，市场中使用以下五类核心资产类型：

- `skill`
- `plugin`
- `trigger`
- `channel`
- `experience`

更具体的边界判断和示例，见 `references/asset-types.md`。

## 水产市场与 `openclawmp` 的关系

`openclawmp` 是与 OpenClaw 水产市场交互的 CLI 工具。

它主要用于：

- 检查本地 CLI 和凭证状态
- 搜索市场资产
- 查看资产详情
- 安装资产
- 列出本地已安装资产
- 卸载资产
- 发布本地资产目录

当用户请求这些具体操作时，应回到 `SKILL.md` 的执行流程，而不是在本文件中展开平台背景。
