## 1. OpenClaw Buddy

**仓库**: https://github.com/RandyChen1985/openclaw-buddy
**作者**: RandyChen1985
**星标**: 26 ⭐（最受欢迎）

### 核心功能
OpenClaw 的**带外管理控制台**（Out-of-Band Management Console），独立于 OpenClaw 主进程运行，提供：

- **系统监控**: CPU/内存实时监控、API 调用统计
- **设备管理**: 双态设备控制、在线状态、远程重启
- **Bot/模型管理**: 动态查看 bots 列表、模型状态、实时刷新
- **消息管理**: 完整消息记录、二维码认证、消息发送
- **自我修复**: 自动备份、SQLite 日志、异常恢复
- **多实例支持**: 管理多个 OpenClaw 实例

### 技术架构
```
[Go + Gin + React] + [WebSocket] + [SQLite]
        ↓
   OpenClaw 实例管理
```

### 关键洞察
1. **OOB 设计模式**：管理控制台独立部署，不占用主进程资源
2. **实时性**：WebSocket 推送所有状态变更
3. **持久化**：SQLite 存储消息、设备状态、备份
4. **安全性**：16 位 BUDDY_TOKEN 认证

### 可复用模式
- **监控代理模式**：独立进程定期检查 OpenClaw 健康状态
- **状态同步机制**：通过 WebSocket 实时推送
- **自我修复**：备份 + 异常检测 + 自动恢复

### 与当前项目关联
当前项目的 `monitor-agent` 技能与之类似，但 Buddy 是**独立服务**（Go + React），功能更完整。

**借鉴**：
- 可以扩展 monitor-agent 为 Web 控制台（当前是 CLI）
- 增加实时监控面板
- 添加设备管理功能

## 2. Claw-auto-coding

**仓库**: https://github.com/Krislu1221/Claw-auto-coding
**作者**: Krislu
**版本**: v3.1
**星标**: 2 ⭐

### 核心功能
基于 **Karpathy 的 Project Manager v2.1** 状态的自动化编码系统，支持：

- **多 Agent 协同**：Coordinator 统一调度，Scratchpad 共享上下文
- **全流程自动化**：需求分析 → 代码生成 → 测试 → 部署
- **状态驱动**：通过 `STATUS.md` 跟踪项目进度
- **预检机制**：执行前检查环境、依赖、前置条件
- **模型协作**：支持 qwen3.6-plus、kimi-k2.5 等多模型协作

### 技术架构
```
用户需求
   ↓
Coordinator (调度器)
   ↓
Scratchpad (共享上下文)
   ↓
┌──────────┬──────────┬──────────┐
│  Analyst │ Coder    │ Tester   │
│ (分析员)  │ (编码员)  │ (测试员)  │
└──────────┴──────────┴──────────┘
   ↓
   STATUS.md (项目状态文件)
```

### 关键洞察
1. **状态文件驱动**：`STATUS.md` 是单点事实（single source of truth）
2. **Scratchpad 模式**：所有 Agent 共享同一上下文，避免信息孤岛
3. **Pre-flight Check**：执行前验证环境，避免中途失败
4. **多模型协作**：不同任务使用不同模型，发挥各自优势

### 与当前项目关联
当前 1688 项目可以借鉴：
- 使用 STATUS.md 跟踪开发进度（当前只有 MEMORY.md）
- 引入 Scratchpad 共享上下文（当前是线性对话）
- 增加预检步骤（如依赖检查、环境验证）

## 5. Skill Optimization Toolkit

**仓库**: https://github.com/shenhu919/openclaw-skill-optimization
**作者**: shenhu919

### 核心功能
OpenClaw 技能优化全流程工具包：

#### 组件 1: skill_cleanup.py
- **去重**：识别重复技能，合并相似功能
- **描述优化**：改进 SKILL.md 的 description 字段
- **技能验证**：检查技能完整性、可运行性
- **质量评分**：综合评分（文档、代码、测试）

#### 组件 2: skill_creation.py (Enhanced Learning)
- **轨迹分析**：分析 5 个历史任务提取模式
- **任务识别**：自动识别任务类型（如"爬虫"、"分析"）
- **模板生成**：基于模式生成新技能模板
- **智能补全**：根据历史类似任务补全步骤

#### 组件 3: skill_categories.md
技能分类体系（5 大类）：
- **A 类**：信息查询（搜索、问答）
- **B 类**：任务处理（转换、计算）
- **C 类**：OpenClaw 系统管理
- **D 类**：文档生成
- **E 类**：特定领域专用

### 关键洞察
1. **数据驱动优化**：基于历史任务日志（`task_history.json`）改进技能
2. **模式提取**：从成功案例中提取通用模板
3. **质量评估**：量化技能质量（文档完整性、测试覆盖）
4. **分类体系**：标准化技能管理

### 与当前项目关联
当前项目已创建多个技能（monitor-agent, self-learning 等），可使用此工具包：
- 运行 `skill_cleanup.py` 检查现有技能质量
- 使用 `skill_creation.py` 基于历史任务自动生成新技能
- 按照分类体系整理技能目录

**行动建议**：
```bash
# 将工具集成到 OpenClaw
cd ~/.openclaw/workspace
git clone https://github.com/shenhu919/openclaw-skill-optimization.git skills/skill-optimization-toolkit
```

## 6. Agent Optimization Framework

**仓库**: https://github.com/nanox2001/agent-optimization-framework
**作者**: nanox2001

### 核心功能
Agent 行为优化与优先级控制框架：

- **控制链配置**：`control-default-chain.json` 定义 Agent 执行顺序
- **引擎优先级**：`engine-priority-control.json` 控制模型选择策略
- **场景映射**：`tool-scenario-mapping.json` 工具-场景匹配
- **自动研究**：Autoresearch 自动研究最优配置

### 关键配置文件
```json
// control-default-chain.json
{
  "chain": ["planner", "researcher", "coder", "reviewer"]
}

// engine-priority-control.json
{
  "priority": ["claude", "gpt4", "stepfun"],
  "fallback": true
}
```

### 关键洞察
1. **可配置的执行链**：Agent 调用顺序可配置，而非硬编码
2. **模型优先级**：根据任务类型选择不同模型
3. **工具-场景匹配**：自动选择最适合的工具
4. **Autoresearch**：自动研究最优参数配置

### 与当前项目关联
当前项目使用固定模型（MiniMax-M2.7 primary），可借鉴：
- 实现任务类型检测，动态选择模型
- 配置多模型 fallback 链
- 工具使用场景优化（如 embedding 用 Ollama，文本生成用 MiniMax）

## 8. Consciousness Engine

**仓库**: https://github.com/michaelyu945945/openclaw-Consciousness-engine
**作者**: michaelyu945945
**语言**: TypeScript

### 核心功能
OpenClaw 认知架构引擎，实现：
- **感知**（Perception）：过滤和优先级排序
- **思考**（Engine）：主动推理循环
- **记忆**（Memory）：模式和上下文持久化
- **学习**（Learning）：从交互中提取模式
- **反思**（Reflection）：自我评估和行为调整

### 架构图
```
┌─────────────────────────────────────────┐
│     Consciousness Engine (意识引擎)     │
├───────────┬───────────┬─────────────────┤
│Perception │  Engine   │    Memory       │
│  (感知)    │ (思考)    │   (记忆)        │
└───────────┴───────────┴─────────────────┘
         │              │
         └──────┬───────┘
                ▼
           Learning (学习)
                │
                ▼
           Reflection (反思)
```

### 核心算法
**Engine.think()** 流程：
1. **理解阶段**：提取用户输入关键信息
2. **回忆阶段**：从记忆库检索相关历史
3. **意图预测**：预判用户下一步需求
4. **策略制定**：生成行动计划

### 关键洞察
1. **认知循环**：感知 → 思考 → 行动 → 学习 → 反思，闭环迭代
2. **主动思考**：不是被动响应，而是主动推理
3. **记忆检索**：基于语义相似度检索相关记忆
4. **自我反思**：从经验中学习，调整未来行为

### 与当前项目关联
当前项目已有：
- `memory-pro`（记忆系统）
- `self-reflection`（周复盘）
- `project-stuck-detect-loop`（困境检测）

**差距**：
- 缺少**主动感知**模块（当前是用户触发）
- 缺少**实时学习**（当前是周度复盘）
- 缺少**意图预测**（当前无预判）

**整合建议**：
将 Consciousness Engine 的认知循环整合到 OpenClaw：
1. 每次用户消息触发 Perception 模块
2. Engine 主动思考（当前 LLM 推理）
3. 行动后 Learning 模块提取模式
4. 定期 Reflection 模块自我评估
