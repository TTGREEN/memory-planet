记忆系统架构，三层设计：

---

**第一层：Event Ingest Layer**
```
对话输入 → memory.js ingest → atoms.db
```
负责把仕泽说的话转成 atom 写入 SQLite。每条 atom 有：
- `content` — 内容文本
- `confidence` — 可信度（写入时定）
- `importance` — 长期价值（每日更新）
- `human_pin` — 手动置顶标志
- `salience` — 此刻相关性（recall 时实时算，不落表）

---

**第二层：Recall Engine**
```
atoms.db → memory.js recall → recall 结果
```
查询时按评分公式混合排序：
- 语义相似度 40%
- 新鲜度 20%
- importance 20%
- salience 10%
- relation_density 10%（M0 没有这类数据，为 0）

recall 结果通过 `memory-recall-inject` 插件注入到 prompt。

---

**第三层：Storage & Sync**
```
atoms.db（主）→ memory/daily-logs/YYYY-MM-DD.md（镜像）
```
atoms.db 是唯一真相来源，daily-logs 单向同步，用于 OpenClaw 搜索 corpus。

---

**当前进度：M0 搭建完成**，正在做 M0.5（recall 自动注入上下文）。M1 开始做 active-memory 召回和 relation/claim 表。
