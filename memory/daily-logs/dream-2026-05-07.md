## 二、最重要的学习（来自源码）

### Hermes Agent 的 Memory Provider 架构

```
memory_manager.py   ← 编排层
       ↑
memory_provider.py  ← ABC，定义接口
       ↑
plugins/memory/     ← honcho.py, mem0.py, openviking.py...
```

**启发**：接口定义 + 插件实现是扩展性的关键。
我的记忆系统也应该定义清晰的接口，具体存储方式可以切换。

### Hermes Agent 的 Curator Lifecycle

后台自动维护 skill 的生命周期：
- stale_after_days: 多久不活跃算 stale
- archive_after_days: 多久后 archive
- backup: 备份策略

**启发**：我的 consolidate-memory.ps1 应该增加"最后修改时间"检查，
超过 N 天没访问的文件自动标记为 stale 并 archive。

### ianlpaterson 的 8 条设计规则

这些规则是用痛苦换来的：

1. Every file must be discoverable via index
2. Every lesson must be date-stamped [YYYY-MM-DD]
3. Every write target must have a fixed schema
4. Every cron job must be budgeted and alert on failure
5. Every index must have a staleness detector
6. Every fact lives in exactly one canonical location
7. Every file must fit its loading mechanism
8. Don't rebuild what the platform provides natively

**启发**：这些规则是我设计决策的 guard rails。
每次写新代码前，应该先问：这个决策符合规则吗？
