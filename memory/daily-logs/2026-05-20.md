## 二、发现问题

### MiniMax API 行为异常

**问题表现：**
- `judgeContradictionWithLLM` 始终返回 `FALSE`
- `generateParadigmShift` 返回空字符串
- `callMiniMax` 响应包含 `thinking` block 但 `text` block 为 `undefined`

**根本原因：**
`callMiniMax` 一直发送 `"thinking_enabled": false`（即使设为 false），而 MiniMax 在收到 `thinking_enabled` 字段时行为不同：
- 发送 `thinking_enabled: false` → 返回 `thinking` block，无 `text` block
- 不发送 `thinking_enabled` 字段（thinking=false 时）→ 返回 `text` block

**修复方案：**
```javascript
// 之前（错误）
body = { model: MODEL, max_tokens, thinking_enabled: thinking, ... }

// 之后（正确）
body = { model: MODEL, max_tokens, ... }
if (thinking) body.thinking_enabled = true;  // 只在 true 时发送
```

### judgeContradiction 检测失败

**问题：** `max_tokens: 10` 太低，输出被截断

**修复：**
- `max_tokens: 10 → 200`
- system prompt 改为单行（减少模型解释空间）
- 多层模式匹配：`^TRUE\W` / `\bTRUE\b` / 中文矛盾关键词

### generateParadigmShift 输出被截断

**问题：** `max_tokens: 150` 不够（thinking 时需更多 token）

**修复：**
- `max_tokens: 150 → 600`
- 增加 fallback：`thinking: true` 失败时用 `thinking: false` 再试

## 七、M5 断点续传架构落地

### 问题：SIGKILL 根因

dream-entropy 跑到一半被系统 SIGKILL，不是 timeout 太短的问题，而是**脚本思维的天然限制**：
- cron 的 agentTurn 在 120s 后强制 kill
- exec 所有输出缓存在内存里，LLM 长文本撑爆 buffer
- 进程树退出时所有子进程一起死

### 解决方案：spawn + SQLite 状态机

仕泽提出"操作系统守护进程思维"——用断点续传 + detached worker 重构。

**evolution_tasks 表：**
```sql
CREATE TABLE evolution_tasks (
  id          TEXT PRIMARY KEY,
  subject     TEXT NOT NULL,
  pair_data   TEXT NOT NULL,   -- JSON pairs
  status      TEXT DEFAULT 'PENDING',  -- PENDING / PROCESSING / COMPLETED / FAILED
  result_data TEXT,
  error_msg   TEXT,
  created_at  TEXT,
  updated_at  TEXT
);
```

**架构：**
```
star-soul-core-runner.js dream-entropy
  ├─ enqueueEvolutionTasks() → 写入 evolution_tasks（所有 subject clusters）
  └─ spawn(node, dream-entropy-worker.js, { detached: true, stdio: pipe })
       └─ child.unref() → 父进程立即返回，worker 独立运行
            └─ 每处理完一 subject 更新 status=COMPLETED
            └─ 中途崩溃/断电 → 重启后从 PENDING 继续（幂等）
```

**关键代码（star-soul-core-runner.js）：**
```javascript
const child = spawn(process.execPath, [workerScript, 'loop'], {
  detached: true,
  stdio: ['ignore', outFd, errFd],  // 输出到文件，不走内存 buffer
});
child.unref();
```

### 新增文件

| 文件 | 用途 |
|------|------|
| `dream-entropy-worker.js` | detached worker，支持 `loop`（处理）和 `enqueue`（入队）两种模式 |
| `migrations/006-evolution-tasks.sql` | evolution_tasks 表 schema |

### cron 集成改造

`memory.js atoms ssc dream-entropy` 改为 detached spawn：
```javascript
const ssc = spawn('node', [runner, 'dream-entropy'], {
  detached: true,
  stdio: 'pipe',
});
ssc.unref();  // 父进程不等待
process.exit(0);  // 立即返回
```


dream-entropy cron job timeoutSeconds: 120s → **30s**（worker 自己管自己）
